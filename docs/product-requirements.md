# BCTrust Product Requirements

## Objective

Build a blockchain project and tokenomics scanner that helps users evaluate Web3 assets using liquidity, holder distribution, contract risk, market activity, and trust indicators.

## Core Requirements

- Users should be able to search for a token by contract address, symbol, or project name.
- The system should display token overview information.
- The system should display liquidity pool information.
- The system should display holder distribution insights.
- The system should display contract verification indicators.
- The system should support project risk categories.
- The system should calculate a trust score concept.
- The dashboard should be simple enough for newer crypto users but useful enough for researchers and analysts.

## Data Requirements

- Token price
- Market cap
- Volume
- Liquidity pool size
- Holder distribution
- Contract address
- Contract verification status
- Ownership status
- Liquidity lock status concept
- Risk indicator flags

## Future Requirements

- Multi-chain token support
- AI-generated risk summaries
- Watchlists
- Alerting
- Exportable risk reports
- Wallet clustering
- Social sentiment scoring
